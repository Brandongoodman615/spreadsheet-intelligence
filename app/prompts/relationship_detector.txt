and sample values.

- "foreign_key" — ID-level key match (e.g. Products.supplier_id → Suppliers.id)
- "lookup"      — value match against a reference table (e.g. Orders.status → Statuses.code)
- "reference"   — semantic/name match (e.g. Pipeline.Owner → Organization.Employee)

- "exact_name"  — column names are identical or near-identical (e.g. both named "category_id")

- to_table/to_column is the "one" or referenced/dimension side
- Be conservative with confidence: use 0.9+ only for exact ID matches, 0.6-0.8 for semantic
- If no relationships exist (e.g. single sheet, or sheets are truly independent), return []
